We created a file named original.txt. Then a hard link and a symbolic link were created. The ls -li command displays the inode numbers and shows that the original file and hard link share the same inode, while the symbolic link has a different inode.

2) Comparison Between Hard Link and Symbolic Link

Hard Link:
• It shares the same inode as the original file.
• It points directly to the file data.
• It continues to work even if the original filename is deleted.

Symbolic Link:
• It has its own inode.
• It stores the pathname of the original file.
• It becomes broken if the original file is deleted.

Observations:
Hard links provide another name for the same file, whereas symbolic links act as shortcuts pointing to another file.

5) Conclusions Drawn from the Experiment

This experiment demonstrates that hard links and symbolic links behave differently. Hard links share the same inode as the original file and continue to work after the original filename has been deleted. Symbolic links have a separate inode and only store the path to the original file, hence they become broken when the original file is removed.
