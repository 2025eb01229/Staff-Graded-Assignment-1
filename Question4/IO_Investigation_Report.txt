1) Open files identified

Commands:
echo "Linux I/O Investigation" > io_test.txt
cat io_test.txt

Output:
Linux I/O Investigation

Observations:
The echo command created a new text file and the cat command displayed its contents. This confirms that the file can be accessed and read successfully.

2) File descriptor observations

Command:
ls -l /proc/$$/fd

Output:
total 0
lrwx------ 1 rijul_bose rijul_bose 64 Jul  2 16:15 0 -> /dev/pts/0
lrwx------ 1 rijul_bose rijul_bose 64 Jul  2 16:15 1 -> /dev/pts/0
lrwx------ 1 rijul_bose rijul_bose 64 Jul  2 16:15 2 -> /dev/pts/0
lrwx------ 1 rijul_bose rijul_bose 64 Jul  2 16:15 255 -> /dev/pts/0

Observations:
Linux uses file descriptors to manage input and output operations. File descriptor 0,1 and 2 represent standard input, standard output and standard error respectively.

3) Output redirection and error redirection results

Commands:
echo "Hello Linux" > output.txt
cat output.txt
ls missingfile 2> error.txt
cat error.txt

Output:
abc

ls: cannot access 'missingfile': No such file or directory

Observations:
The > operator redirects normal output to a file. 
2> redirects error messages to a separate file. 
This keeps standard output and errors organized for easy access and use.

5) Explanation of how Linux manages file I/O

Linux manages file I/O using file descriptors. Standard input, standard output and standard error are represented by the file descriptors 0, 1, and 2 respectively. Redirection operators allow us to store command output or error messages in files instead of displaying them on the terminal making them accessible and reusable if and when needed.
